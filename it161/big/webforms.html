HTML Forms

Forms exist in physical forms like on a paper and in digital forms, like on a website. When developing and designing a website, it is crucial to add HTML form elements such as text boxes, check boxes, radio buttons, click buttons, lists, etc. These forms are important because they help you gather data and information from your audience and inform them, also it leaves your audience with a great user experience!

Maneet Puri, a web consultant says,”A web form often acts as a communication bridge that allows a user to communicate with a company and vice versa. Illogical and inappropriate questions can hamper the company image quite easily. For example, you want the users to create an account with your website. Asking for a residential address would be quite irrelevant. The first thing that comes to his mind will be “Why do you need my residential address for just a sign-up?”. This not only causes confusion, but also distrust and chances are good that people don’t go through with the signup at all.” (USABilla) In other words, it is a good business practice to have organized and appealing forms that ask only what is needed and gives out useful information and offers assistance.

So how do you create these forms and how do you make them look pretty? You need HTML to begin with and you style them with CSS. Supun Kavinda has this great blog post that teaches you how to style forms with CSS. He explains the process in 6 steps:

  1. Setting box-sizing
  2. CSS selectors for input elements
  3. Basic styling methods for text input fields
  4. Styling other input types
  5. UI pseudo-classes
  6. Non customizable inputs(LogRocket)

But how do you style HTML forms using CSS, without HTML? That’s why you need to learn the HTML form fundamentals. WebPlatform has a good guide on the HTML fundamentals and has writtens a basic HTML form code to help explain, as you can see below:

          <form>
          Name: <input type="text" name="name" id="name" value="" />
          Email: <input type="text" name="email" id="email" value="" />
          Comments: <textarea name="comments" id="comments" cols="25" rows="3"></textarea>
          <input type="submit" value="submit" />
          </form>

And they have explanations for each element and attributes, such as The </form> element is required to wrap around the form’s content — without it you don’t have a web form. The </form> tag can have a few attributes. Note that you can’t nest a form inside another form. (WebPlatform) You can check out more, by visiting their website at
